## Context

제조 AI MLOps 플랫폼에서 고객사별 Agent 웹앱 요청이 빠르게 늘어났습니다.  
프로젝트마다 요구사항은 조금씩 달랐지만, 입력 폼/실행 상태/결과 표시 같은 흐름은 반복됐습니다.

## Problem

- Agent 프로젝트가 3개 이상으로 늘어나면서 동일한 UI/로직을 프로젝트별로 재구현하면서 유지보수 비용이 커졌습니다.
- 수정 사항이 생기면 여러 레포에 같은 변경을 반복 반영해야 했습니다.
- 신규 프로젝트 온보딩 시 구현 기준이 분산되어 품질 편차가 발생했습니다.

## Options

1. 프로젝트별 독립 구현 유지
2. 화면 단위로 공통 컴포넌트만 추출
3. 반복 인터랙션을 기준으로 컴포넌트 + Hook + 상태 규약을 패키지화

프로젝트별 차이는 남기되 반복 흐름을 구조적으로 흡수하기 위해 3번을 선택했습니다.

## Solution

- 공통화 대상: 입력 검증 흐름, 실행 상태 전이, 결과 패널 패턴
- 분리 전략: `도메인별 확장 포인트`와 `반복 인터랙션 핵심`을 분리
- 배포 방식: 별도 패키지로 관리해 신규 프로젝트에서 즉시 재사용

```ts
// 예시: 공통 상태 전이 패턴
type AgentRunState = "idle" | "running" | "success" | "error";
```

## Results

- 3개 이상 Agent 프로젝트에서 반복되던 동일 수정 작업을 공통 패키지로 흡수
- 프로젝트 간 코드 재사용률 약 60% 달성
- 신규 Agent 프로젝트 초기 화면 구성 시간을 단축
- 공통 모듈 기준으로 협업해 코드 리뷰/품질 기준이 일관화

## What I'd do next

- 공통 패키지의 버전 정책과 변경 로그를 강화해 팀 간 의존성 충돌을 줄입니다.
- 반복 패턴 탐지를 자동화해 공통화 후보를 더 빠르게 발견합니다.
