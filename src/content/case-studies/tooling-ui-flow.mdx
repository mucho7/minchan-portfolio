## Context

여러 단계를 순서대로 수행해야 하는 도구형 UI에서, 기능은 충분했지만 사용자가 현재 맥락과 다음 행동을 놓치기 쉬웠습니다.

## Problem

- 기능이 나열된 구조라 작업 순서가 화면에서 명확히 드러나지 않았습니다.
- QA와 신규 개발자가 동일한 시나리오를 재현하는 데 시간이 오래 걸렸습니다.
- 결과 확인 방식이 분산되어 실패 원인을 추적하기 어려웠습니다.

## Options

1. 기존 UI 유지 + 문서 보강
2. 화면 배치만 부분 수정
3. 상태 전이 중심으로 단계 UI/피드백 체계를 재설계

학습 비용을 화면 자체에서 줄이기 위해 3번을 선택했습니다.

## Solution

- 단계별 상태를 명시적으로 노출하고, 다음 행동을 버튼/가이드 문구로 연결
- 결과 영역을 수행 맥락과 가까운 위치로 이동해 피드백 루프 단축
- 기획/디자인과 상태별 색상 규칙 및 예외 시나리오를 합의해 일관성 확보

```ts
const stepLabels = ["입력", "검증", "실행", "결과"];
const isNextActionEnabled = currentStep.isValid && !isRunning;
```

## Results

- 복잡한 기능의 이해 비용을 문서 의존이 아닌 UI 자체에서 흡수
- 신규 사용자 온보딩 속도 개선
- QA-개발 간 상태 해석 불일치 감소

## What I'd do next

- 단계별 이탈 지표를 수집해 막힘 구간을 정량적으로 추적합니다.
- 사용성 테스트를 통해 안내 문구와 피드백 타이밍을 더 정교화합니다.
