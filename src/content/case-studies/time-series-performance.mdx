## Context

CNC 관제 대시보드에서 대량 시계열 데이터를 탐색하는 화면을 담당했습니다.
사용자는 확대/이동/구간 비교를 반복 수행하기 때문에, 체감 성능이 곧 업무 생산성이었습니다.

## Problem

- 5천만 건 이상 데이터에서 차트 라이브러리가 임의로 데이터를 생략하거나 다운샘플링해 탐색 맥락이 깨졌습니다.
- 상태 구독 범위가 넓어 작은 변경에도 렌더링 비용이 과도하게 발생했습니다.
- 사용자가 탐색 흐름을 유지하기 전에 UI 지연이 먼저 발생했습니다.

## Options

1. 라이브러리 기본 최적화 옵션만 사용
2. 서버 전처리만 강화
3. 클라이언트 다운샘플링 + 상태 구독 분리 + 렌더링 경량화 결합

탐색 맥락을 화면에서 즉시 유지해야 했기 때문에 3번을 선택했습니다.

## Solution

- LTTB 기반 다운샘플링으로 해상도 대비 정보 손실을 최소화
- 화면 상태와 데이터 상태를 분리해 불필요한 재렌더 범위를 축소
- 고빈도 이벤트 구간에서 렌더링 파이프라인을 단순화

```ts
const TARGET_FRAME_MS = 16;
const shouldDownsample = totalPoints > visibleWidth * 2;
```

## Results

- 대량 데이터 구간에서도 60FPS 수준 인터랙션을 목표로 성능 안정화
- 탐색/비교/재탐색 흐름에서 지연 체감이 크게 감소
- 성능 개선을 렌더링 튜닝이 아닌 상태·데이터·표현 계층 설계 문제로 재정의

## What I'd do next

- 데이터 분포별로 다운샘플링 전략을 자동 전환해 정확도와 속도 균형을 개선합니다.
- 시나리오 기반 성능 회귀 테스트를 추가합니다.
